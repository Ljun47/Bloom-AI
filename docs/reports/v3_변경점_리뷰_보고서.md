## 2. 발견된 이슈 (심각도별)

### Critical (즉시 조치 필요)

#### C-1. `asyncio.gather()` 부분 실패 시 데이터 불일치

**파일**: `src/api/routes/podcasts.py` (라인 186-196)

```python
await asyncio.gather(
    backend_client.save(RESOURCE_PODCAST_METADATA, episode_request),
    backend_client.ingest_podcast_episodes(...),
)
```

**문제**: `return_exceptions=True` 미사용. 한 작업이 먼저 완료되고 다른 작업이 실패하면:
- `podcast_metadata`는 DB에 저장됨
- `podcast_episodes`는 실패
- 전체 `gather`가 예외를 발생시켜 사용자에게 500 에러 반환
- 하지만 DB에는 이미 부분 데이터 존재

**권장 조치**:
```python
results = await asyncio.gather(
    backend_client.save(RESOURCE_PODCAST_METADATA, episode_request),
    backend_client.ingest_podcast_episodes(...),
    return_exceptions=True,
)
for i, result in enumerate(results):
    if isinstance(result, Exception):
        logger.warning("저장 작업 %d 실패: %s", i, result)
```

#### C-2. `content_analyzer.py` BackendClient 리소스 누수

**파일**: `src/agents/podcast/content_analyzer.py` (라인 126-137)

```python
try:
    backend_client = BackendClient()
    await backend_client.ingest_mind_frequencies(...)
    await backend_client.close()  # ← 예외 발생 시 도달 불가
except Exception as e:
    self.logger.warning(...)      # ← close()가 실행되지 않음
```

**문제**: `ingest_mind_frequencies()` 내부에서 예외가 발생해도 `close()`가 호출되지 않아 `httpx.AsyncClient` 연결 누적.

**권장 조치**:
```python
try:
    backend_client = BackendClient()
    try:
        await backend_client.ingest_mind_frequencies(...)
    finally:
        await backend_client.close()
except Exception as e:
    self.logger.warning(...)
```

#### C-3. `.env` 파일 `BACKEND_API_URL` 충돌

**파일**: `.env` (라인 52)

```
BACKEND_API_URL=http://localhost:8080/api
```

**문제**: `config/loader.py`의 기본값은 `http://localhost:8080/greenroom/ingest/ai`로 변경되었으나, `.env` 파일은 구 경로(`/api`)를 여전히 설정 중. `dotenv`가 로드되면 `.env` 값이 우선하여 **모든 save/load API가 잘못된 URL로 호출됨**.

**영향**: 로컬 개발 환경에서 모든 내부 저장 API 호출 실패 가능.

**권장 조치**: `.env` 파일의 `BACKEND_API_URL` 값을 업데이트하거나 주석 처리:
```
BACKEND_API_URL=http://localhost:8080/greenroom/ingest/ai
```

#### C-4. `.env` 파일 `PROMPT_DIR=/app/prompts` 테스트 영향

**파일**: `.env` (마지막 줄)

```
PROMPT_DIR=/app/prompts
```

**문제**: `config/loader.py`가 `load_dotenv()`를 모듈 레벨에서 호출하여 `PROMPT_DIR=/app/prompts`가 설정됨. 로컬 개발 환경에서는 `/app/prompts` 경로가 존재하지 않아 **모든 프롬프트 로딩이 실패**. 이로 인해 테스트 56개가 실패 중.

**권장 조치**: `.env` 파일에서 `PROMPT_DIR` 주석 처리 (컨테이너 전용 설정):
```
# PROMPT_DIR=/app/prompts  # 컨테이너 환경 전용 — 로컬에서는 기본값 "prompts" 사용
```
