**발표자**: AI 담당

## 📄 페이지 1 — AI가 만들어낸 차별점

저희 팀이 만든 **Mind-Log**는 초개인화 AI 멘탈케어 플랫폼입니다. 사용자의 감정과 생각을 **12개의 AI 에이전트**가 분석하여, 개인에게 맞춘 **멘탈케어 팟캐스트와 시각화 이미지**를 제공합니다.

일반적인 챗봇 서비스와 저희가 다른 점은 네 가지입니다. 첫째, 12개 에이전트를 **Phase 0부터 4까지 계층적으로 오케스트레이션**해 책임을 분리했습니다. 둘째, **병렬 Fan-out 구조**로 분석 에이전트들이 동시에 실행되어 응답시간을 크게 단축했습니다. 셋째, **CRISIS 선점 메커니즘**을 통해 위기 신호가 감지되면 다른 생성 작업을 즉시 중단하고 안전 응답을 우선합니다. 넷째, **Self-Reflection 품질 검증** 단계를 두어 0.863의 검증 정확도를 확보했습니다.

현재 **583개의 pytest 통과**로 구현 안정성을 담보하고 있습니다.

> 자수: 약 400자

## 📄 페이지 2 — Phase 기반 AI 파이프라인

이 슬라이드는 저희 시스템의 **핵심 구조**입니다. **Phase 0**의 Intent Classifier가 사용자 의도를 분류하고 1차 위기 감지를 수행합니다. **Phase 1**은 병렬 Fan-out 구조로 Safety, Emotion, Content Analyzer, Podcast Reasoning 네 개의 에이전트가 **동시에 실행**됩니다.

이 과정에서 Reasoning 에이전트는 필요할 때만 **독립 RAG 에이전트**인 Episode Memory와 Knowledge Agent를 조건부로 호출하여 개인 맥락과 전문 지식을 불러옵니다. **Phase 2**에서는 대본 생성과 시각화가 병렬 진행되고, **Phase 3**의 Batch Validator가 5단계 자기검증을 거쳐 기준 미달 시 재생성을 명령합니다. 마지막 **Phase 4**는 사용자 톤에 맞춘 개인화입니다.

여기서 가장 중요한 것은, Safety가 **CRISIS**를 감지하는 순간 Phase 1 전체 병렬 실행이 **즉시 취소**된다는 점입니다. 안전을 구조로 보장한 설계입니다.

> 자수: 약 410자

## 📄 페이지 3 — AI 기술 스택 & 성과

기술 스택은 역할에 맞춘 **이중 LLM 전략**이 핵심입니다. 고추론이 필요한 Safety, Reasoning, Personalizer에는 **Sonnet**을, 고속 분류가 필요한 Emotion, Content, Script, Validator에는 **Haiku**를 배치해 비용과 지연을 동시에 최적화했습니다.

오케스트레이션은 **LangGraph StateGraph**로 Phase 기반 워크플로우를 선언적으로 관리하고, AgentState TypedDict로 에이전트 간 **필드 소유권을 분리**했습니다. 멀티모달은 **Stable Diffusion과 LoRA**로 감정 기반 커버 이미지를 생성하며, 추론 과정은 Chain-of-Thought와 Graph-of-Thought 두 전략을 결합했습니다.

RAG는 **Pinecone에 에피소드 메모리와 전문 지식을 분리 저장**하고, **Neo4j에 GoT 추론 그래프**를 누적하는 이중 레이어 구조입니다. 결과적으로 **0.863 검증 정확도**, **583개 테스트 통과**, **12 AI 에이전트** 완성이라는 성과를 달성했습니다.

> 자수: 약 400자

## 📄 페이지 4 — 개발 중 마주친 과제와 해결

마지막으로 개발 과정에서 마주친 **세 가지 핵심 과제와 해결**을 말씀드립니다.

**첫째, LLM 할루시네이션과 품질 편차 문제**입니다. Batch Validator의 **Self-Reflection 5단계 검증**과 임계값 0.75 기준의 **재시도 최대 2회** 구조를 도입해 평균 0.863의 검증 정확도를 확보했습니다.

**둘째, AWS Bedrock 쓰로틀링 문제**입니다. **Exponential Backoff**로 재시도 간격을 점진적으로 늘려 API 회복 여유를 확보했고, **Circuit Breaker**로 연속 실패 시 해당 구간을 즉시 격리하여 장애가 전체 파이프라인으로 번지지 않도록 했습니다.

**셋째, 프롬프트 인젝션과 PII 유출 보안 이슈**입니다. **Input · Output 이중 Sanitizer**를 모든 에이전트의 공용 인프라로 **설계 단계부터 내재화**해 사후 조치가 아닌 구조적 방어를 구현했습니다.

저희 팀의 방향은 **빠르게 만드는 AI가 아니라, 안전하고 검증된 AI를 구조로 설계**하는 것이었습니다. 감사합니다.

> 자수: 약 420자
